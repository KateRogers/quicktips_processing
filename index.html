Overview of Data Processing

01

Data Management Process

A data management process is an approach to working with data from the planning of a data exercise to destruction or retention of the data gathered.

No matter the scope of the data exercise, working with data always involves a series of non-linear steps that help us systematically define and plan our goals, gather data, prepare and analyse that data and ultimately use it to drive data-informed actions.

01

The Goal of Data Processing

Data processing is an umbrella term that refers to the methods for collating and manipulating data to meet analytical requirements. Data processing is all the steps required to take data from its raw, often messy, form and prepare it for analysis. Data procesing includes:

  1. verifying data: The process of checking data for accuracy, authenticity and reliability;
  2. cleaning data: The process of correcting and standardising data from a record set, table, or database.
  3. transforming data: The process of reshaping data to facilitate processing and analysis; and
  4. merging data: The process of combining data from different files, worksheets and tables.

01

Messy COVID-19 Data

With so much data being shared about the pandemic, we often want to jump in and find the insight. But remember, pristine data is very rare, and COVID-19 data is no exception. Before we analyse, we have to process messy COVID-19 data because:

  1. There may be missing data or errors in the data.
  2. The data may come in an odd shape, (e.g. a ‘wide’ data format).
  3. Data sources may be reporting different numbers of cases due to their different appraoches to collection.
  4. Critical data points for analysis will be stored in different files, (e.g. the number of tests and the confirmed cases) and need to be brought together.
  5. Different names for locations or date formats may be used.

01

Start with Data Verification

Before analysing data, it is important to understand what the data in your spreadsheet is actually measuring. That is why we always check the metadata, assess the collection methodology and read through the data dictionary first. This way we understand what is and is being represented by the data in our spreadsheet.

01

Next to Start Data Cleaning

Data cleaning is the process of correcting and standardising data from a record set, table, or database. It broadly consists of detecting, correcting, replacing, modifying or removing messy data. If often feels like the work of data cleaning is never done.

Remember. before doing anything, back up the original data. Then, your can start editing and removing messy data for our file with peace of mind.

01

What is Data Validation?

Data validation is essentially the steps we take to ensure that the data we have is in a standardised format and is meaningful. There are many types of data validation checks we could perform, for example:

  1. A presence check to ensure there is no missing data;
  2. A format checks to ensure that data is in a standardised format, e.g. YYYY-MM-DD;
  3. A type check to ensure the correct data type has been entered, e.g. confirmed cases is a number not a string; and
  4. A range check to ensure that a value falls within the specified or meaningful range, e.g. the number of confirmed cases is positive;

01

Validating COVID19 Data

In the Johns Hopkins University confirmed cases dataset, we would run the following data validation checks.

  1. We do a presence check to see if for each column there is data entered;
  2. We do a type check to ensure that the data is in a numeric format and not text format;
  3. We do a format check to ensure that the dates are using the desired date convention, in this case YYYY-MM-DD; and
  4. We do a range check to verify that we only have data on the desired time period and that the case data is positive.

02

Transforming & Merging Data

Data validation, verifying and cleaning help to ensure that the data we have is good enough for our analysis. However, even once our data is ‘good enough’, there is more processing work to be done. We will often have to reshape and bring together different datasets in order to do our analysis.

  1. Transforming data involves converting data from one format or structure to another. In this section, you'll learn about data strcutures and how to transform data from wide to long and long to wide based on your needs.
  2. Merging data involves bringing two or more data sources together. We do this in order to compare data more easily as well as to complete it when the data we need sits in different files. In this section, you'll learn how to merge data using lookup formulas.

02

Tabular Data Basics

In tabular data every column of a table represents a particular variable and each row corresponds to a given record, or observation.

  1. A variable contains all values that measure the same attribute or characteristic.
  2. A record, or observation, contains all values measured on it.

This is the data structure required for creating charts like line graphs in spreadsheet software.

What is Long Data?

Long data is, as the name suggests, longer with more rows and less columns. Each value sits in an own row meaning there is a row of data for each country on each day we have data for. This format is also sometimes referred to as narrow, tall, stacked or tidy data. While it can be less ‘human-readable’, it is often more straightforward to work with and analyse data in this format.

02

Merging Data?

Merging data, as the name implies, is the process of combining two or more datasets into a single dataset. Data that you may want to analyse will often be stored in multiple files, worksheets, or data tables and we have to bring it together in order to analyse it all in one go. We will often merge data to:

  1. Compare different datasets. We may need to compare the data we have with other sources in order to triangulate and detect errors while we are processing. Or, during analysis, we may want to compare data to see relationships, trends, patterns, etc.
  2. Complete the data we have. Many datasets we use are collected around a single topic or theme. To incorporate multiple themes or facets of a topic into our data analysis, we need to bring multiple pieces of data together.

02

When to Merge Data

Merging small amounts of data can be performed by copying and pasting but this is time consuming, not reproducible and error prone. Consider a more sophisticated method for merging datasets if you find yourself:

  1. Resizing windows side-by-side to visually compare values in separate spreadsheets.
  2. Copying and pasting multiple values from one spreadsheet to another.
  3. Re-typing values from one spreadsheet to another to match-up to a similar variable.

02

Using Lookup Formulas

If you are using Excel, you may want to use a lookup formula to merge data - VLOOKUP, HLOOKUP or XLOOKUP. Once you get a hang of LOOKUP formulas, you will quickly and easily be able to merge data, wasting less time and minimising the chance of error.

  1. Lookup_value: The value we want to lookup and should be the same in the both datasets.
  2. Lookup_Array: The column where you want to look up the lookup value.
  3. Return_Array: The column that you want to return when you find the lookup value.

02

We are going to use the XLOOKUP formula to add two additional data points, the Inform COVID-19 Risk score and class, to our dataset of COVID19 confirmed cases and confirmed deaths in locations with a Humanitarian Response Plan.

  2. Lookup_Array: The column that contains the ISO3 code in the Inform dataset (column b)
